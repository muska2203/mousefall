### Параллельные задачи
- **1, 2, 3** можно делать **параллельно** (они не зависят друг от друга).
- **4** зависит от **2** и **3**, но может начаться, как только они готовы.
- **5** зависит от **1, 2, 3, 4** — последняя из simulation-задач.
- **6** (UI) можно делать параллельно с **5**, так как зависит только от **4**.
- **7** (интеграция) — финальная, после готовности **1–5**.

## Правила работы с задачами

1. Каждая задача — самодостаточный документ. Не нужно переключаться между файлами, чтобы понять, что делать.
2. В задаче указаны **все файлы**, которые нужно менять, и **код**, который нужно написать.
3. В задаче указаны **тесты** с примерным кодом.
4. После выполнения задачи запускаем `npm run typecheck` и `npm test`.
5. Если задача ломает существующие тесты — это баг, нужно исправить.
