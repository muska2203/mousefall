   308	## Key Architectural Decisions
   309	
   310	### 1. Mutable State with Controlled Access
   311	
   312	Игровое состояние мутируется напрямую внутри Simulation. Внешние слои получают доступ только через `Readonly<GameState>` или ViewModel от Presentation.
   313	
   314	**Tradeoff:** Менее "чисто", чем immutable-подход, но гораздо проще в отладке и разработке.
   315	
   316	### 2. Presentation as the Sole API Consumer
   317	
   318	Только Presentation знает о существовании Simulation API. UI работает с абстракцией Presentation.
   319	
   320	**Tradeoff:** Дополнительный слой кода для простых действий, но зато UI полностью заменяем и независим от игровой логики.
   321	
   322	### 3. Direct Function Calls (No Command Pattern)
   323	
   324	Игровые действия — plain functions, не объекты-команды. Достаточно для одиночной игры.
   325	
   326	**Tradeoff:** Сложнее добавить replay/undo позже, но сейчас значительно проще.
   327	
   328	### 4. Seeded RNG in Simulation State
   329	
   330	Вся случайность через seeded PRNG, хранимый в `GameState`.
   331	
   332	**Tradeoff:** Чуть сложнее `Math.random()`, но критично для детерминизма.
   333	
   334	### 5. JSON Content with Runtime Validation
   335	
   336	Контент в JSON, валидация при загрузке через Zod.
   337	
   338	**Tradeoff:** Нет compile-time проверки контента, но можно редактировать без пересборки.
   339	
   340	### 6. Action → Intent → Event Tree
   341	
   342	Simulation использует трёхфазную систему:
   343	- **Action** — высокоуровневое намерение (MOVE, ATTACK, WAIT)
   344	- **Intent** — низкоуровневые операции после разрешения (MOVE, DAMAGE, DIE)
   345	- **Event** — неизменяемая запись о произошедшем, организованная в дерево `ExecutionNode`
   346	
   347	Presentation потребляет дерево `ExecutionNode` и превращает его в анимационные планы.
   348	
   349	---
   350	
   351	## Determinism Contract
   352	
   353	Simulation layer **должна** быть детерминированной:
   354	- Одно начальное состояние + одна последовательность действий = один результат
   355	- Вся случайность только через seeded RNG
   356	- Нет `Date.now()`, `Math.random()`, async-операций в Simulation
   357	- Порядок обработки сущностей консистентен (сортировка по ID)
   358

   360	
   361	## Testability Contract
   362	
   363	Simulation layer **должна** тестироваться без браузера:
   364	- Нет DOM-зависимостей
   365	- Нет browser API
   366	- Чистые функции где возможно
   367	- Контролируемые мутации состояния с чёткими точками входа
   368	
   369	Presentation layer тестируется отдельно:
   370	- Мокается Simulation API
   371	- Проверяется логика перевода событий UI → команды Simulation
   372	- Проверяется формирование анимационных планов из `ExecutionNode`

## Минимальный контекст проекта Cats and Mouse

### Что это
2D пошаговый roguelike на TypeScript в браузере. Ядро игры (simulation) реализовано и работает. Presentation и UI — запланированы, но не реализованы.

### Техстек
- TypeScript 5.5+, Vite, Vitest (Node.js, без браузера)
- React 18 + PixiJS 8 — установлены, но пока не используются
- Zod для валидации JSON-контента

---

### 4 архитектурных слоя (строгие правила зависимостей)

```
UI           → presentation/, utils/constants.ts
Presentation → simulation/ (только публичный API), utils/
Simulation   → content/, utils/
Content      → (ничего)
utils/       → (ничего)
```

**Главное правило:** Presentation — единственный слой, который имеет право вызывать API Simulation. UI не знает о существовании Simulation.

| Слой | Ответственность | Статус |
|------|----------------|--------|
| **Content** (`public/content/`) | JSON-данные: враги, предметы, карты, способности | Работает |
| **Simulation** (`src/simulation/`) | Headless детерминированная игровая логика. Action → Intent → Event | Работает |
| **Presentation** (`src/presentation/`) | Мост UI↔Simulation, сессионное состояние, анимационные планы | **Пустая директория** |
| **UI** (`src/ui/`) | React-компоненты, PixiJS-рендерер, ввод, анимация | **Только README.md** |

---

### Ключевые правила

- **Детерминизм:** Simulation использует только seeded RNG (`utils/rng.ts`), никогда `Math.random()`. Одно состояние + одни действия = один результат.
- **Тестируемость без браузера:** Simulation не использует DOM, browser API, сетевые запросы, async.
- **Мутация состояния:** только внутри Simulation. Внешние слои получают `Readonly<GameState>` или ViewModel.
- **Контент без пересборки:** JSON в `public/content/` валидируется через Zod при загрузке.
- **Нет циклических зависимостей.**
- **Язык:** вся документация, комментарии, markdown — на русском. Код (переменные, типы) — на английском.
