# Правила слоя UI

> Работая в `src/ui/`, соблюдай эти правила. Они приоритетнее общих.

---

## Критически важно

- **Только отрисовка** — рендеринг мира и интерфейса.
- **Только ввод** — захват клавиатуры, мыши, тач-событий.
- **Только анимация** — исполнение анимационных планов от Presentation.
- **Не мутировать игровое состояние напрямую.**
- **Не содержать игровую логику.**
- **Не импортировать `simulation/` напрямую** — только `presentation/`.

---

## Структура внутри UI

```
src/ui/
  ├── renderer/       # PixiJS — отрисовка мира, спрайтов, тайлов
  ├── components/     # React — HUD, меню, панели, инвентарь
  └── input/          # Обработчики ввода
```

> Renderer (PixiJS) — **не отдельный архитектурный слой**, а техническая подсистема внутри UI.

---

## Ответственность

- Получать ViewModel + AnimationPlan от Presentation и отрисовывать
- Отправлять события ввода в Presentation (не принимать решений)
- Исполнять анимации через PixiJS и сообщать Presentation о завершении
- Единственный слой с доступом к browser API (DOM, localStorage)

---

## Частые задачи

| Задача | Куда идти |
|--------|-----------|
| Добавить React-компонент | `components/` |
| Добавить экран | `screens/` |
| Добавить анимацию спрайта | `renderer/` или `animation/` |
| Добавить обработку ввода | `input/` |
| Изменить стили | `styles/` |

---

## Разрешённые зависимости

- `src/presentation/` (получение ViewModel, отправка событий ввода)
- `src/utils/constants.ts` (TILE_SIZE и визуальные константы)
