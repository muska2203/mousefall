# Архитектура боевой системы на основе реакций

## Основная идея

Не делать способности монолитными. Вместо этого вся логика игры строится
из универсальных правил вида:

``` text
Когда
↓
Если
↓
То
```

Каждый предмет, талант, статус, оружие и пассивка представляет собой
набор таких правил.

## Примеры

### Оружие

**Меч**

``` text
Когда используется намерение «Удар»
→ Нанести режущий урон
```

**Молот**

``` text
Когда используется намерение «Удар»
→ Толкнуть цель на 2 клетки
```

### Статусы

**Горение**

``` text
Когда начинается ход
→ Получить урон
```

**Кровотечение**

``` text
Когда актер перемещается
→ Потерять здоровье
```

### Артефакты

``` text
Когда наложен статус
→ Получить 1 ОД
```

``` text
Когда получены ОД
→ Следующая атака усилена
```

Из этого автоматически возникает цепочка:

``` text
Удар
↓
Кровотечение
↓
StatusApplied
↓
+1 ОД
↓
Следующая атака усилена
```

Никто специально не описывал эту комбинацию --- она возникает благодаря
общим правилам.

## Статусы как состояния мира

Статусы не обязаны что-либо делать сами по себе.

Например, статус **Мокрый** становится строительным блоком:

-   Молния → двойной урон по мокрой цели.
-   Лед → замораживает мокрую цель.
-   Огненный урон / горение → убирает статус «Мокрый».
-   Огненный урон / горение + Мокрый → создают пар (fog).

## Намерения

Игрок выбирает намерение (например, «Толкнуть»), а все остальное
определяется правилами:

-   Молот → толчок на 2 клетки.
-   Перчатки → толчок вызывает кровотечение.
-   Амулет → толчок создает искры.
-   Талант → удар о стену оглушает.
-   Оглушение → увеличивает следующий урон.

Одна кнопка запускает длинную цепочку реакций.

## Реакции второго порядка

Эффекты сами создают новые события.

``` text
Нанесен урон
↓
DamageEvent
↓
Урон > 10
↓
Создать ударную волну
↓
Новый DamageEvent
↓
Поджечь масло
↓
FireCreated
↓
Дым
↓
Ослепление
```

Игрок создает условия, а система сама развивает ситуацию.

## Универсальная структура правила

``` text
Когда
↓
Кому
↓
Что сделать
↓
При каких условиях
```

Например:

-   Когда нанесен урон.
-   Кому: цели.
-   Действие: наложить кровотечение.
-   Если цель горит.

## Ограничения и порядок

Чтобы избежать хаоса:

-   контентные реакции выполняются перед системными мировыми реакциями;
-   порожденные контентными реакциями интенты выполняются рекурсивно до запуска системных реакций на исходное событие;
-   порядок правил: `source` → `target` → `world` → `radius`, затем `priority`, затем `ruleId`;
-   для модификаторов внутри одного слоя сначала применяются множители, затем линейные `+`/`-`;
-   правило срабатывает один раз на объект события, но может срабатывать многократно в рамках одной цепочки, если происходит несколько подходящих событий;
-   запрещены бесконечные циклы реакций (лимит на глубину цепочки);
-   правила статуса, наложенного реакцией в процессе цепочки, активируются только после её завершения;
-   игроку показывается журнал цепочки реакций (см. детали визуализации в
    [`docs/plans/Презентационный_слой_и_UI.md`](./Презентационный_слой_и_UI.md)).
