## 0. Архитектурные решения

1. **`DisplayState`** — минимальная модель для UI: карта (tiles, visible, explored), сущности (id, x, y, hp, maxHp, type, templateId, isAlive, statusEffects), игрок, мета (floor, round, turnSide, phase).
2. **`DisplayPatch`** — неизменяемая запись об изменении `DisplayState`, порождённая одним `GameEvent`.
3. **`PresentationNode`** — объединяет `GameEvent`, `DisplayPatch` и `AnimationNode[]` (опционально), а также флаг `isFieldAnimation` для FOV-фильтрации.
4. **Патчи применяются по мере завершения анимаций.** `AnimationSequencer` уведомляет Presentation о завершении каждого шага; Presentation применяет соответствующий `DisplayPatch`.
5. **После завершения всех анимаций фазы** `DisplayState` ресинкается с финальным `GameState` — перестраховка на случай расхождений.
6. **FOV-фильтрация** применяется только к полевым анимациям, но не к событиям/патчам/логу.
7. **Подчистка костылей:** убрать старые блокировки отображения состояний (хп-баров, статусов, позиций спрайтов, камеры и т.д.), которые были нужны из-за рендеринга финального `GameState` поверх анимаций. Конкретные места определяет агент при выполнении шагов 2 и 4.

  - `buildDisplayState(state: GameState): DisplayState` — начальное состояние;
  - `createPatch(event: GameEvent): DisplayPatch` — маппинг событий в патчи;
  - `applyPatch(state: DisplayState, patch: DisplayPatch): DisplayState` — иммутабельное применение.
- [x] Создать `src/presentation/displayState/sync.ts`:
  - `resyncDisplayState(state: GameState): DisplayState` — ресинк после анимаций.
- [x] Создать `src/presentation/displayState/planner.ts`:
  - `buildPresentationPlan(result: SimulationResult, state: GameState): PresentationNode[]` — обход дерева `ExecutionNode`, построение патчей и анимаций, FOV-фильтр только для полевых анимаций.

### Тесты
- [x] `tests/unit/presentation/displayState/builder.test.ts`:
  - `buildDisplayState` строит корректное состояние;
  - `createPatch` возвращает ожидаемый патч для каждого значимого `GameEvent`;
  - `applyPatch` корректно обновляет позиции, HP, статусы, карту, видимость;
  - `resyncDisplayState` синхронизирует с финальным `GameState`.

### Примечание
Для событий, которые не меняют визуальное состояние (например, `RULE_TRIGGERED`, `ACTION_APPLIED` в большинстве случаев), использовать `NO_OP` патч.
