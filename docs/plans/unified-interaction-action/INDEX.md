# Декомпозиция задачи: единый action взаимодействия с объектами

Эта папка содержит логические блоки реализации плана [`UNIFIED_INTERACTION_ACTION.md`](../UNIFIED_INTERACTION_ACTION.md).

## Принципы декомпозиции

- Каждый блок — отдельный файл с полным контекстом и инструкциями.
- Блоки не содержат ссылок на конкретные строки кода, чтобы не требовалась актуализация после изменений.
- Каждый блок имеет чёткую входную точку (что нужно знать) и результат (что получится).
- Блоки выполняются последовательно. Зависимости указаны в каждом блоке.

## Порядок выполнения

```
Блок 1 → Блок 2 → Блок 3 → Блок 4 → Блок 5 → Блок 6
```

| Блок | Название | Короткое описание |
|---|---|---|
| [BLOCK_01_BASE_ACTION](./BLOCK_01_BASE_ACTION.md) | Базовый action `INTERACT` | Добавить тип action, `interactionKind`, `resolveInteraction`, handler, регистрацию, AP cost. |
| [BLOCK_02_FLOOR_ITEM_CONTAINER](./BLOCK_02_FLOOR_ITEM_CONTAINER.md) | Контейнер предмета на полу | Заменить `ItemEntity` на полу на `FloorItemContainerEntity`. |
| [BLOCK_03_INTENT_EXECUTION](./BLOCK_03_INTENT_EXECUTION.md) | Выполнение intent'ов из `INTERACT` | Научить `INTERACT` порождать конкретные intent'ы и создать `FLOOR_TRANSITION`. |
| [BLOCK_04_PRESENTATION](./BLOCK_04_PRESENTATION.md) | Presentation и UI | Перевести подсказки, автопуть и `moveOrAttack` на `INTERACT`. |
| [BLOCK_05_REMOVE_AUTO_STAIR](./BLOCK_05_REMOVE_AUTO_STAIR.md) | Удаление авто-спуска по лестнице | Убрать `stairsTransitionReaction` и связанные intent/event. |
| [BLOCK_06_CLEANUP](./BLOCK_06_CLEANUP.md) | Удаление старых action'ов | Удалить `PICKUP`, `OPEN_DOOR`, `CLOSE_DOOR`, `DESCEND`, `ASCEND` и обновить документацию. |

## Зависимости между блоками

- **Блок 2** зависит от **Блока 1**, потому что `FloorItemContainerEntity` использует `interactionKind: 'item'`, который `resolveInteraction` должен уметь обрабатывать.
- **Блок 3** зависит от **Блоков 1 и 2**, потому что `INTERACT` должен уметь порождать intent'ы для дверей, контейнеров предметов и лестниц.
- **Блок 4** зависит от **Блока 1** минимально. Для полноценного тестирования нужны **Блоки 2 и 3**.
- **Блок 5** зависит от **Блока 3**, потому что переход по лестнице должен работать через `INTERACT` + `FLOOR_TRANSITION` до удаления авто-спуска.
- **Блок 6** зависит от всех предыдущих блоков.

## Рекомендация по выполнению

После каждого блока запускать тесты. Не переходить к следующему блоку, пока текущий не стабилен.
