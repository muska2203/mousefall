# SYNC_STATUS — Статус рассинхрона документации и кода

> Этот файл помогает агенту понимать, каким системам можно доверять, а где источник правды — код.

---

## Как пользоваться

- Если система `[STABLE]` — код должен соответствовать документации.
- Если система `[DRAFT]`/`[EXPERIMENTAL]` — документация может отставать; источник правды — код + тесты.
- Если нашёл новый рассинхрон — добавь строку в таблицу.
- После устранения рассинхрона — удали строку.
