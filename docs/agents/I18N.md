# I18N — Локализация

> **Статус:** `[STABLE]` — система локализации устаканилась.
> **Источник правды:** этот файл.

> Весь пользовательский текст в игре проходит через систему локализации. Хардкод строк запрещён.

---

## Стек

- **i18next** + **react-i18next** — ядро локализации.
- **Статические TS-ресурсы** — переводы в `*.ts`, не в JSON. Type-safe через `satisfies Resources`.
- **Два языка** — `ru` (fallback) и `en`.
- **Язык переключается** через `useSettingsStore` → `localStorage` → `i18next.changeLanguage`.

---

## Структура

```
src/i18n/
  config.ts          # Инициализация i18next + загрузка ресурсов
  schema.ts          # Единая схема типов Resources (все namespace и ключи)
  types.ts           # Вспомогательные типы
  hooks.ts           # useTranslation wrapper
  t.ts               # t() для кода вне React (Presentation / Simulation)
  locales/
    ru/              # Русские переводы
      common/        # Общие строки (yes, no, hp, xp)
      screens/       # Экраны (mainMenu, characterCreation, game, ending)
      components/    # Компоненты (heroPanel, inventoryPanel, ...)
      system/        # Presentation / Simulation (logBuilder, itemMapper, ...)
      index.ts       # Агрегатор ruResources
    en/              # Английские переводы (зеркало ru/)
      index.ts       # Агрегатор enResources

src/content/texts/
  types.ts           # Типы для игрового контента
  ru.ts              # Тексты врагов, предметов, способностей (русский)
  en.ts              # Тексты контента (английский)
  lookup.ts          # getContentText(category, id, locale)
```

---

## Namespace

| Namespace | Что содержит | Где использовать |
|-----------|--------------|------------------|
| `common` | Общие UI-строки (`yes`, `no`, `hp`, `xp`, `level`) | Любой компонент через `useTranslation('common')` |
| `screens` | Заголовки, кнопки, подписи экранов | Экраны: `MainMenuScreen`, `GameScreen`, ... |
| `components` | Панели, тултипы, labels | Компоненты: `HeroPanel`, `InventoryPanel`, ... |
| `system` | Combat log, мапперы, валидации, названия эффектов | `presentation/`, `simulation/` |

## Как добавить строку в UI (React)

```tsx
import { useTranslation } from '@i18n/hooks';

// 1. Выбери правильный namespace
const { t } = useTranslation('components');

// 2. Используй ключ БЕЗ префикса namespace
<span>{t('heroPanel.title')}</span>

// 3. Если нужен другой namespace в том же компоненте — второй хук
const { t: tc } = useTranslation('common');
<span>{tc('game.hp')}</span>
```

**Важно:** `useTranslation('components')` привязывает namespace. `t('common.game.hp')` **не сработает** — будет искать ключ `common.game.hp` внутри `components`.

## Как добавить строку в коде вне React (Presentation / Simulation)

```ts
import { t } from '@i18n/t';

// Ключ ВКЛЮЧАЕТ namespace, разделитель — точка
const label = t('system.gameSession.equipSlotWeapon');
const text = t('system.logBuilder.heroAttacked', { target: 'Кот' });
```

`t()` из `@i18n/t` автоматически разбирает `namespace.key.subKey` по первой точке.

## Как добавить игровой контент с текстом

**JSON-шаблоны НЕ содержат текста.** Они описывают только механику:

```json
{
  "id": "health_potion",
  "type": "consumable",
  "rarity": "common"
}
```

Все тексты (name, description, flavorText) живут в `src/content/texts/`:

```ts
// src/content/texts/ru.ts
export const ruContentTexts: ContentTexts = {
  items: {
    health_potion: {
      name: 'Зелье здоровья',
      description: 'Восстанавливает 30 HP.',
    },
  },
  // ...
};
```

Чтобы получить локализованный шаблон:

```ts
import { getLocalizedItem } from '@content/registry';
const template = getLocalizedItem('health_potion', 'ru');
// template.name === 'Зелье здоровья'
```

## Как добавить новый ключ в схему

1. **Добавь ключ в `src/i18n/schema.ts`** — в соответствующий interface.
2. **Добавь перевод в `src/i18n/locales/ru/...`** — и зеркало в `en/...`.
3. **Используй ключ в коде.**
4. **Запусти `npm run typecheck`** — схема гарантирует, что все языковые файлы синхронизированы.

### Правила именования ключей

- `camelCase`.
- Точечная нотация для вложенности: `gameSession.heroStatStrength`.
- Файл перевода не должен превышать **150 строк** — разбивай на подфайлы при необходимости.

## Что НЕЛЬЗЯ делать

- ❌ Хардкодить русский/английский текст в JSX или в `console.log`.
- ❌ Добавлять `name` / `description` / `flavorText` в JSON-шаблоны контента.
- ❌ Использовать `t('common.game.hp')` из хука `useTranslation('components')`.
- ❌ Забывать добавлять английский перевод при добавлении русского.
