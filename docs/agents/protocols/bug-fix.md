# Протокол: bug_fix

## Когда применять

Исправление багов, падений тестов, неожиданного поведения.

---

## Последовательность действий

1. **Воспроизведи проблему.**
   - Найди или напиши тест, который падает.
   - Если теста нет — создай минимальный reproduction case.

2. **Локализуй причину.**
   - Читай код, а не только документацию.
   - Используй [`SYNC_STATUS.md`](../SYNC_STATUS.md), чтобы понять, каким системам можно доверять.

3. **Предложи минимальное исправление.**
   - Не рефакторь смежный код.
   - Если баг затрагивает архитектуру — остановись и обсуди.

4. **Реализуй и проверь.**
   - Запусти упавший тест.
   - Запусти связанные тесты.
   - Запусти `npm run typecheck`.

5. **Финал.**
   - Сообщи, что было сломано и как исправлено.
   - Если документация устарела из-за исправления — обнови или отметь в [`SYNC_STATUS.md`](../SYNC_STATUS.md).
