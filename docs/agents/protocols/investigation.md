# Протокол: investigation

## Когда применять

Разбор «как работает», «почему так», поиск причины проблемы без немедленного исправления.

---

## Последовательность действий

1. **Уточни вопрос пользователя.**
   - Что именно нужно понять?
   - Нужен ли код ответа или достаточно объяснения?

2. **Читай источники правды в правильном порядке:**
   - [`SYNC_STATUS.md`](../SYNC_STATUS.md) — чтобы понять, чему доверять.
   - Документацию `[STABLE]`-систем.
   - Код `[DRAFT]`/`[EXPERIMENTAL]`-систем.
   - Тесты для примеров поведения.

3. **Не меняй код и документацию**, пока пользователь не попросит.

4. **Финал.**
   - Дай краткий ответ: как работает, где ключевые файлы, какие риски/нюансы.
   - Если нашёл рассинхрон — отметь его для [`SYNC_STATUS.md`](../SYNC_STATUS.md).
