# Протокол: docs_update

## Когда применять

Обновление, актуализация или создание документации.

---

## Последовательность действий

1. **Уточни цель обновления.**
   - Актуализировать под текущий код?
   - Зафиксировать новое решение?
   - Добавить рецепт/протокол?

2. **Проверь источник правды.**
   - Если документируешь стабильную систему — проверь по коду и тестам.
   - Если документируешь нестабильную — не делай вид, что всё устаканилось; используй `[DRAFT]`.

3. **Обнови документацию.**
   - Держи файлы компактными.
   - Разделяй «must know» и «deep dive».
   - Обновляй [`SYNC_STATUS.md`](../SYNC_STATUS.md), если рассинхрон устранён.

4. **Финал.**
   - Сообщи, какие файлы обновлены.
