# Протокол: refactor

## Когда применять

Переписывание, упрощение, вынесение кода без изменения поведения.

---

## Последовательность действий

1. **Убедись, что поведение не должно измениться.**
   - Если рефакторинг меняет семантику — это `system_design`, а не `refactor`.

2. **Найди или создай тесты, фиксирующие текущее поведение.**
   - Рефакторинг без тестов рискован.

3. **Сделай минимальное изменение.**
   - Один шаг за раз.
   - Не смешивай рефакторинг с новыми фичами.

4. **Запусти тесты после каждого шага.**
   - `npm run typecheck`
   - `npm test`

5. **Финал.**
   - Сообщи, что изменено.
   - Укажи, прошли ли все тесты.
