## Что уже решено

- Скиллы остаются как сейчас. У каждого оружия должно быть минимум три скилла: **атакующий**, **вспомогательный**, **защитный**.
- Скиллы должны быть **тупыми генераторами интентов**: проверяют цель, создают простые интенты и проставляют нужные теги. Сложные комбо реализуются через реакции.
- Мировые реакции делятся на два слоя:
  - **системные** — код, инварианты мира (смерть, столкновения, тики статусов);
  - **контентные** — `ContentRuleReaction`, data-driven правила от предметов, талантов, статусов и окружения.
- Две фазы выполнения правил:
  - **модификаторы на интенте** — до `IntentExecutor`;
  - **реакции на событии** — после `IntentExecutor`.
- Порядок модификаторов: **сначала множители, потом линейные `+` / `-`**.
- Глубина цепочки реакций ограничена, например, **1000 реакций за шаг**.
- Правила собираются из четырёх источников: **источник действия**, **цель действия**, **мир / окружение**, **сущности в радиусе события**.
- Правила кэшируются у актора в `activeRules` и добавляются / удаляются при надевании / снятии экипировки, наложении / снятии статусов, получении / отзыве талантов.
