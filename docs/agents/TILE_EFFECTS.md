# Работа с тайловыми эффектами

> **Статус:** `[STABLE]` — инструкция по добавлению тайловых эффектов актуальна.
> **Источник правды:** этот файл для процесса; `src/simulation/systems/intents/` и `src/simulation/content-rules/` для кода.

> Этот файл — инструкция для агентов. Архитектурное описание системы см. в [`../architecture/TILE_EFFECTS.md`](../architecture/TILE_EFFECTS.md).

---

## Быстрый обзор

Тайловые эффекты — это динамические материалы на клетках карты: вода, масло, огонь и т.п. Они живут в `public/content/tile-effects/` как JSON-шаблоны, а своё игровое поведение получают через контентные правила и статусы.

Есть два вида объектов:

- **Тайловый эффект** — сам материал (`water`, `oil`).
- **Статус тайлового эффекта** — временное состояние материала (`burning` на масле).

Чтобы добавить, например, горящее масло, нужно создать и то, и другое, а затем связать их правилами.

---

## Добавление нового тайлового эффекта

В качестве примера используется уже реализованная пара `water` / `oil`.

### Шаг 1. Создать JSON-шаблон

Создайте файл `public/content/tile-effects/<идентификатор>.json`. Имя файла должно совпадать с полем `id` внутри.

Основные поля шаблона:

- `layer` — на текущем этапе всегда `cover`.
- `duration` — базовая длительность в ходах.
- `renderOrder` — порядок отрисовки относительно других эффектов на той же клетке.
- `ruleIds` — идентификаторы контентных правил, которые активируются при событии на клетке с эффектом.
- `blockedByTileEffects` — типы эффектов, при наличии которых данный не накладывается.
- `mutuallyExclusiveWithTileEffects` — типы эффектов, которые заменяются данным при наложении.
- `canHaveStatus` — статусы тайловых эффектов, которые можно наложить на этот материал.
- `durationDecreasesWhenHasStatus` — если эффект должен исчезать только при определённом статусе (как масло сгорает только при `burning`).

### Шаг 2. Добавить тексты

Добавьте локализованное название в `src/content/texts/ru/tile-effects.ts` и `src/content/texts/en/tile-effects.ts`. Ключ в объекте — идентификатор эффекта, значение — объект с полем `name`. Без записи шаблон не получит локализованного имени, но игра не сломается: будет fallback на идентификатор.

### Шаг 3. Добавить спрайт

Поместите файл спрайта в `public/assets/tile-effects/<идентификатор>.png`. Рендерер использует соглашение об именовании: путь строится как `/assets/tile-effects/<effectType>.png`.

### Шаг 4. Зарегистрировать в манифесте

Добавьте путь к JSON-файлу в массив `tileEffects` в `public/content/manifest.json`. Без этого шаблон не загрузится.

### Шаг 5. Добавить контентные правила

Если материал должен что-то делать при входе актора, при получении урона или при тике:

1. Добавьте правило в массив `CONTENT_RULES` в `src/simulation/content-rules/rules.ts`.
2. Укажите идентификатор правила в поле `ruleIds` шаблона тайлового эффекта.

При событии на клетке с эффектом правило автоматически попадёт в мировой слой `tileEffect`.

Глобальные мировые правила, не привязанные к конкретному эффекту, добавляются в массив `GLOBAL_WORLD_CONTENT_RULES` в `src/simulation/content-rules/world-rules/global-rules.ts`.

Полезные условия и селекторы:

- `inTileEffect` — проверяет, что на клетке есть указанный эффект.
- `tileEffectHasStatus` — проверяет, что у эффекта на клетке есть указанный статус.
- `tilesInRadius` — выбирает соседние клетки с указанным эффектом (нужен для распространения).

### Шаг 6. Добавить способность или предмет (опционально)

Если эффект должен появляться в обычной игре:

1. Создайте шаблон способности в `public/content/abilities/<id>.json`.
2. Создайте исполнитель способности в `src/simulation/skills/executors/<id>Skill.ts`, который порождает интенты `SPAWN_TILE_EFFECT`.
3. Зарегистрируйте исполнитель в реестре скиллов.
4. Добавьте анимацию применения в `src/presentation/animation/skills/`.
5. Добавьте текст и спрайт для UI.

### Шаг 7. Написать тесты

Добавьте тесты в одну из папок:

- `tests/unit/simulation/intents/tile-effect-intent-executor.test.ts` — для проверки исполнителей интентов.
- `tests/unit/simulation/content-rules/<правило>.test.ts` — для проверки отдельных правил.
- `tests/integration/tile-effects/<сценарий>.test.ts` — для сквозных сценариев.

Проверьте edge cases: замена через `mutuallyExclusiveWithTileEffects`, блокировка через `blockedByTileEffects`, обновление длительности при повторном спавне, удаление вместе со статусами.

### Шаг 8. Запустить проверки

После изменений запустите: `npm run typecheck`, затем `npm test`, затем `npm run validate:content`.

## Добавление нового статуса тайлового эффекта

Если материал должен временно менять своё поведение (гореть, электризоваться, замерзать), добавьте статус тайлового эффекта.

### Шаг 1. Создать JSON-шаблон статуса

Создайте файл `public/content/tile-effect-statuses/<идентификатор>.json`:

- `duration` — базовая длительность.
- `ruleIds` — правила, активируемые, когда статус присутствует на клетке события.
- `statusCategory` и `categoryPriority` — для разрешения конфликтов.
- `mutuallyExclusiveWith` и `blockedBy` — конфликты с другими статусами тайловых эффектов.
- `renderOrder` — порядок отрисовки. Обычно статусы рисуются поверх базового материала.
- `neverExpires` — если статус не должен тикать по своей длительности (как `burning` на масле, который гаснет только вместе с маслом).

### Шаг 2. Разрешить статус у материала

Добавьте идентификатор статуса в поле `canHaveStatus` шаблона(ов) тайлового эффекта, на которых он может существовать. Без этого исполнитель наложения статуса отклонит интент.

### Шаг 3. Добавить тексты и спрайт

- Локализованное название — в `src/content/texts/{ru,en}/tile-effect-statuses.ts`.
- Спрайт — в `public/assets/tile-effects/<идентификатор>.png` (рендерер использует ту же директорию, что и для материалов).

### Шаг 4. Зарегистрировать в манифесте

Добавьте путь к JSON-файлу в массив `tileEffectStatuses` в `public/content/manifest.json`.

### Шаг 5. Добавить правила

Статусы тайловых эффектов обычно реагируют на события `ENTITY_MOVED`, `ENTITY_DAMAGED`, `TILE_DAMAGED`, `TILE_EFFECT_STATUS_TICKED` и другие.

- Используйте условие `tileEffectHasStatus` для проверки наличия статуса на клетке.
- Для распространения на соседние клетки используйте селектор `tilesInRadius` с `targetConditions`, проверяющими наличие нужного материала и отсутствие статуса.

### Шаг 6. Протестировать

Убедитесь, что:

- статус накладывается только на разрешённые материалы;
- статус тикает и удаляется корректно;
- при удалении родительского эффекта статус удаляется без отдельных событий;
- `neverExpires` работает ожидаемо.

---

## Частые ошибки

- **Забыли добавить `canHaveStatus`.** Статус не наложится, даже если правило сработает.
- **Забыли `ruleIds` в шаблоне.** Правило существует, но не активируется на клетке с эффектом.
- **Статус не добавлен в манифест.** Игра не загрузит шаблон и не сможет его найти.
- **Спрайт положили не в ту папку.** Рендерер ищет спрайты только в `public/assets/tile-effects/`.
- **`neverExpires` без `durationDecreasesWhenHasStatus`.** Материал никогда не исчезнет, если статус не снимается другим способом.
