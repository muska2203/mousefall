# PROTOCOL — Поведение агента в проекте Mousefall

> Перед каждой задачей прочитай этот файл, определи тип задачи и следуй соответствующему протоколу из `docs/agents/protocols/`.

---

| Тип | Ключевые слова / ситуации | Протокол |
|---|---|---|
| `content_addition` | «добавь врага/оружие/броню/амулет/расходник/способность/статус/карту» | [`protocols/content-addition.md`](./protocols/content-addition.md) |
| `system_design` | «сделай так, чтобы…», «новая механика», «добавь интент/реакцию/Action», «как работает AI» | [`protocols/system-design.md`](./protocols/system-design.md) |

---

## Общие правила

- Не придумывай требования. Если формулировка неоднозначна — задай уточняющий вопрос.
- Делай минимальные изменения. Не рефакторь смежный код без явного разрешения.
- Всегда запускай тесты, относящиеся к изменённой области.
- При противоречии документации и кода:
  - прочитай [`SYNC_STATUS.md`](./SYNC_STATUS.md);
  - если система `[STABLE]`, код должен соответствовать доке;
  - если система `[DRAFT]`/`[EXPERIMENTAL]`, источник правды — код + тесты.
- Если задача затрагивает 3+ системы — используй двухфазный режим: сначала план/дока, потом код.

---

## Когда обязательно остановиться и спросить

- Задачу можно решить несколькими архитектурно разными способами.
- Изменение нарушает принципы из [`LAYERS.md`](./LAYERS.md).
- Тесты падают после изменений и причина неочевидна.
- Документация и код противоречат друг другу, а [`SYNC_STATUS.md`](./SYNC_STATUS.md) не помогает.
- Ты не уверен в типе задачи.
